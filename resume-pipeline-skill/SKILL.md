# 求职全流程助手 (Resume Pipeline)

整合 jd-hunter（JD 猎取）和 resume-matcher（简历生成）两个子 skill，提供统一的求职入口。根据用户目标，自动选择正确路径并逐步引导。

## 入口：意图识别

收到用户请求后，首先确认目标范围。向用户提问（单选）：

> 你想要：
> **A)** 只找岗位 — 搜索 JD、评分、结构化
> **B)** 只写简历 — 用已有 JD 生成简历和扩展材料
> **C)** 完整流程 — 先找岗位，选感兴趣的，再生成简历

**隐式推断**：若用户已隐含答案（如"帮我在北京找后端岗位"→ A），直接确认而不用展示菜单。

---

## 精简交互原则

只在关键决策节点停下，其余步骤自动推进。**不问可以不问的问题**。合理可推断的用默认值。

| 节点 | 触发条件 | 默认策略 |
|------|----------|----------|
| 目标确认（A/B/C） | 入口 | 隐式推断，否则提问 |
| 目标岗位确认 | LLM 推断后 | 展示让用户确认/修改 |
| 搜索参数（城市/经验） | 用户未提供 | 设为空（全国搜索） |
| 评分阈值 | Phase A 开始时 | 默认 7 分，仅用户主动要求时询问 |
| JD 选择（Phase C） | Phase A 结束后 | 展示 index 供选择 |
| 模板文件 | Phase B/C 需要时 | 必问 |
| 个人信息 | 任何 Phase 开始 | 必问 |
| 编写逻辑文件 | Phase B/C | 可选，默认跳过 |
| 扩展材料配置 | Phase B/C | 默认全部 4 种，不询问 |

---

## Phase A：只找岗位

委托给 **jd-hunter**。

1. **收集个人信息**：要求用户提供个人信息文件（须含 `姓名：...`、技能、经历等）
2. **加载子 skill**：`skill("jd-hunter")`
3. **推断目标岗位**：按 jd-hunter 流程推断，将结果反馈用户确认或修改
4. **确认搜索参数**：若用户未指定，询问城市和经验年限（可选，留空则全国搜索）
5. **搜索与评分**：默认阈值 7 分，搜索→评分→结构化
6. **报告结果**：展示各渠道 JD 数量、评分分布、`jd/index.md` 路径

**Phase A 结束后**：若用户未指定 C，询问"要直接用这些 JD 生成简历吗？"— 选择是则进入 Phase C。

### Handoff Note（若后续进入 Phase C）

```
【Handoff: jd-hunter → resume-pipeline】
- 状态：jd/ 目录下有结构化 JD
- 已有：jd/index.md
- 已有：个人信息文件路径
```

---

## Phase B：只生成简历

委托给 **resume-matcher**。用户已有 JD。

1. **收集 JD**：询问 JD 文件路径或目录（单个 `.md` 文件，或含多个 `.md` 的目录）
2. **收集模板**：要求提供简历模板（`.md`，含 `<!-- fill:字段名 -->` 标签）
3. **确认个人信息**：要求提供个人信息文件，提取姓名。若失败则询问用户
4. **可选：编写逻辑**：询问是否有编写逻辑文件，默认跳过
5. **加载子 skill**：`skill("resume-matcher")`
6. **Handoff Note**：

```
【Handoff 到 resume-matcher】
- 跳过：step 1（JD 已由用户提供）
- 已有：JD 文件: （路径）
- 已有：个人信息: （路径）
- 已有：简历模板: （路径）
- 从 step 3（接收模板）继续
```

7. **生成简历与扩展材料**：按 resume-matcher 流程，逐个字段生成填充内容 + 组装简历 + 生成扩展材料（默认全部 4 种）
8. **输出**：`./resume/{姓名}-{岗位}-{公司}.md` 及 `*-extras.md`

### 单 JD vs 批量

- **单 JD**：直接生成一份简历 + 扩展材料
- **批量（JD 目录）**：遍历目录下所有 `.md`，为每个 JD 生成对应简历

---

## Phase C：完整管线（找岗位 → 选岗位 → 写简历）

Phase A + 用户选择 + Phase B。

1. **执行 Phase A**（如上），获取 JD 列表
2. **用户选 JD**：Phase A 结束后，展示 `jd/index.md` 索引。让用户选择感兴趣的 JD：

   ```
   jd/ 目录下有 N 个结构化 JD：
   1. 字节跳动-后端开发.md (匹配度 8/10)
   2. ...
   请输入要生成简历的 JD 序号（逗号分隔，或输入"全部"）：
   ```

3. **收集模板**：要求提供简历模板。若 Phase A 中已收集到个人信息，则复用无需再问
4. **加载子 skill**：`skill("resume-matcher")`，并传递 Handoff Note：

```
【Handoff 到 resume-matcher（Phase C）】
- 跳过：step 1（JD 已在 jd/ 目录下）
- 已有：选中的 JD 列表: （文件路径列表）
- 已有：个人信息: （路径）
- 已有：简历模板: （路径）
- 从 step 3（接收模板）继续
```

5. **逐 JD 生成**：对选中的每个 JD，读取 → 提取字段 → 生成 fill_mapping → 组装简历 → 生成扩展材料
6. **输出**：全部到 `./resume/` 目录
7. **汇总报告**：列出所有生成的简历和扩展材料文件路径

## Handoff Note 协议

### 格式

三段式结构，在加载子 skill 后立即执行：

```
【Handoff 到 <子 skill 名>（<场景说明>）】
- 跳过：<需要跳过的子 skill 步骤编号及原因>
- 已有：<上下文键值对，每行一个>
- 从 <步骤编号> 继续
```

### 优先级

Handoff Note 是编排 skill 的指令，**优先级高于**子 skill 的默认交互。加载子 skill 后，先执行 Handoff Note 指定的跳转和上下文注入，再启动子 skill 流程。

### 使用场景

| 场景 | Handoff 内容 |
|------|-------------|
| Phase A→C 续接 | 跳过 jd-hunter（已完成），标记 jd/ 目录和已收集的个人信息 |
| Phase B 加载 resume-matcher | 跳过 step 1（用户已提供 JD），传递 JD/模板/个人信息路径 |
| Phase C 加载 resume-matcher | 跳过 step 1（jd-hunter 已产出 JD），传递选中 JD 列表 |

## 异常处理

| 场景 | 处理方式 |
|------|----------|
| 用户无个人信息文件 | 提示创建，提供示例格式（须含 `姓名：XXX`） |
| 用户无简历模板 | 提示创建，描述 `fill` 标签格式：`<!-- fill:字段名 -->内容<!-- /fill:字段名 -->` |
| JD 搜索结果为空 | 建议扩大搜索范围（去掉城市/经验限制、换关键词）、或直接提供 JD 文件 |
| 选中的 JD 生成失败 | 跳过该 JD，继续生成下一个，最后汇总失败列表 |
| 子 skill 加载失败 | 告知用户，退回到手动指引 |
| 中间步骤中断 | 下次触发时询问是否从断点继续 |
| 个人信息缺少姓名 | 提示用户手动提供姓名 |
| JD 缺少岗位名称/公司 | 提示用户补充 |

---

## 文件约定

| 文件类型 | 要求 | 格式 |
|----------|------|------|
| 个人信息 | 须含 `姓名：XXX`、技能、经历 | `.md` 或 `.txt` |
| 简历模板 | 含 `<!-- fill:字段名 -->内容<!-- /fill:字段名 -->` 标签 | `.md` |
| JD 文件 | 须含 `岗位名称：XXX` 和 `公司：XXX` | `.md` |
| 输出目录 | 自动创建 | `./resume/` |
| JD 目录 | jd-hunter 输出，resume-matcher 读取 | `./jd/` |

---

## 与其他 Skill 的关系

```
resume-pipeline（本 Skill）
    │
    ├── 委托 ──▶ jd-hunter     （岗位搜索、评分、结构化）
    │
    └── 委托 ──▶ resume-matcher（简历生成、扩展材料）
```

两个子 skill 各自独立可用。resume-pipeline **仅提供统合入口和流程引导**，不包含任何子 skill 的功能代码。
